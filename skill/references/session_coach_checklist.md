# SESSION COACH 진단 체크리스트 (MODE 1용)

> 사용자가 "이 세션 토큰 진단" 또는 "왜 토큰을 많이 써"라고 하면 이 표를 기준으로 점검한다.
> 관찰 신호 → 해당 패턴 ID → 즉시 fix 명령.

| 5 | **수천 줄 로그·에러를 통째로 붙여넣었거나 붙여넣으려 함** | H2-02 로그 전체 덤프 / H8-02 stdout 홍수 | 에러 핵심 20~30줄만 선별해 붙이기. 파일 저장 후 경로 전달. |

| 8 | **"다시 짜줘" / "처음부터 해줘" 요청 / 전체 파일 재출력이 응답** | H6-01 전체 재생성 요청 | "35~42줄에만 수정. 나머지는 그대로." 식으로 범위 명시. |

## 빠른 자가 진단 스크립트 (사용자에게 물어볼 질문 6개)

Claude가 다음 6가지를 사용자에게 확인하면 주요 낭비를 빠르게 잡을 수 있다.

```
1. 지금 세션에서 작업 주제가 몇 개인가? (1개 초과 → H1-01 / H1-04)
2. 같은 파일을 이 세션에서 2회 이상 읽었나? (예 → H2-01)
3. 마지막으로 /clear 또는 /compact 한 게 언제인가? (누적 ~50K 토큰 / 30~40분+ 장기 세션 → H1-03)
4. CLAUDE.md가 있는가? 있다면 200줄 이하인가? (없거나 200줄+ → H7-01 / H3-01)
5. 탐색 요청("파악해줘", "분석해줘")을 메인 세션에서 직접 시켰나? (예 → H8-01 / H5-02)
6. 거대 디렉토리(`node_modules/`·`.next/`·`dist/`) 접근을 `settings.json` `permissions.deny`로 막아뒀나? (안 막았으면 → H3-04)
```

## 패턴별 즉시 fix 커맨드 레퍼런스

| 패턴 | 즉시 fix 커맨드 |
|------|----------------|
| H1-01, H1-04 | `/clear` → 새 세션 시작 |
| H1-02 | `/compact "실패 제외"` 또는 `Esc+Esc`(`/rewind`) |
| H1-03 | `/compact [포커스]` |
| H2-01 | 재읽기 요청 차단, 컨텍스트 내 참조 유도 |
| H2-02, H8-02 | 에러 30줄 선별 또는 PreToolUse 훅 필터 설정 |
| H3-01 | CLAUDE.md 200줄 이하 감량, 불필요 섹션 Skill로 분리 |
| H3-02 | `/mcp`로 미사용 서버 비활성화 (MCP schema는 기본 deferred — 호출·초기화 비용 절감) |
| H3-04 | `settings.json` `permissions.deny`로 거대 디렉토리 차단 (`Read(./node_modules/**)`, `Read(./.next/**)`, `Read(./dist/**)`); `.gitignore`도 일부 존중됨. `.claudeignore`는 미구현이니 의존 금지 |
| H4-01 | `/effort low` (thinking 토글 단축키는 환경별 상이, `/config` 확인) |
| H4-02 | "1줄로만" + `/effort low` |
| H5-01 | `@파일경로` + 완료 기준 명시로 프롬프트 재작성 |
| H5-02, H8-01 | "서브에이전트로 탐색해줘" |
| H5-03 | 요구사항 일괄 명세 또는 SPEC.md 패턴 |
| H5-04 | 출력 범위 명시 ("2줄로만") / CLAUDE.md에 기본값 설정 |
| H6-01 | 수정 범위(줄 번호) 명시 |
| H6-02 | 구현 요청에 테스트 실행 포함 / `/code-review` 커맨드 |
| H7-01 | `/init` → CLAUDE.md 작성 |
| H7-02 | `.claude/skills/`에 반복 패턴 skill로 추출 |
| H7-03 | "ANALYSIS.md로 저장해줘" → 새 세션에서 `@` 참조 소비 |
